艳阳的.NET新世界：关于虚拟布局引擎

虚拟布局引擎是一个多层次、网格化、可嵌套、易于描述的UI布局描述引擎。

所谓网格化，是指一个m行n列的窗口矩阵（1行n列的网格可以理解为有n个选项卡的选项卡窗口），而可嵌套是指矩阵中的每个格可以动态嵌套多个网格层，运行时只有一层可见。

这里，“虚”是指某个布局在设计时可能是不确定的，并且可以在运行时动态生成，由于不同的场景、权限等因素，可能会进行不同的动态匹配。实际上，柔性布局是多层网格动态变化的结果。

艳阳的.NET新世界：关于虚拟Eclipse引擎

虚拟Eclipse引擎是Eclipse项目的动态链接库版本

它包含Eclipse项目的所有代码。其目标是使windows桌面应用程序按照一致的方案实现内置的Eclipse模型， 使宿主桌面软件自然成为Eclipse的启动程序。

与传统的eclipse不同的是，与VEE（虚拟eclipse引擎）集成的应用程序有自己的应用程序对象模型， 由于eclipse部分的所有功能都是基于dll模式提供的，因此VEE事先不知道主机应用程序是什么，宿主应用程序 包含什么对象模型，VEE解决了主机模型和eclipse标准模型之间的连接问题，因此这里的“虚拟”意味着只有在定义 了特定的主机应用程序之后才能确定真正的eclipse模型。同时，宿主应用程序的OSGi模型将比标准eclipse的OSGi模型 更大。换句话说，宿主应用程序系统将提供标准eclipse中没有提供的应用模型部分。

宿主应用程序的实际OSGi模型可能包括不属于标准eclipse OSGi模型的成员，这意味着宿主应用程序同时支持标准eclipse组件 和基于应用程序的eclipse组件，并且在标准浏览器模式下没有基于应用程序的eclipse组件。由于宿主应用程序内置了Eclipse项目 的所有功能，应用程序开发人员可以基于Eclipse技术为宿主应用程序开发新的应用程序内容，从而使宿主应用程序成为一个基于Java技术 的软件生态系统。

艳阳的.NET新世界：关于虚拟浏览器引擎

虚拟Web浏览器引擎是Google Chromium项目的动态链接库版本

VBE包含Chromium项目的所有代码。其目标是使Windows桌面应用程序按照一致的方案实现内置的现代浏览器模型，使宿主桌面软件自然成为现代web浏览器。 与传统浏览器不同的是，与VBE（虚拟Web浏览器引擎）集成的应用程序有自己的应用对象模型，由于浏览器部分的所有功能都是基于dll模式提供的， VBE事先并不知道宿主应用程序是什么，也不清楚宿主应用程序包含什么对象模型，VBE解决了主机模型和浏览器标准模型之间的连接问题，因此这里的“虚拟”是 指只有在定义了特定的主机应用程序之后才能确定真正的浏览器模型。

宿主应用程序的DOM模型将是标准浏览器的DOM模型的一个超集。换句话说，宿主应用程序系统将提供标准浏览器中无法提供的应用模型部分。 宿主应用程序的实际DOM模型可能包括不属于标准浏览器DOM模型的成员，这意味着宿主应用程序同时支持标准网页和基于应用程序的网页 （标准浏览器模式下没有基于应用程序的网页）。由于宿主应用程序内置了Chromium项目的所有功能，应用程序开发人员可以基于Web技术为 宿主应用程序开发新的应用程序内容，使宿主应用程序成为基于Web技术的软件生态系统。

网格化布局：Grid

Grid refers to a grid structure with m rows and n columns. As a container grid, each child grid can carry a specific programmable object ...

网格布局是指一个m行n列的窗口矩阵，做为一个容器，网格中的某个具体的格可以承载新的网格，或者承载一个可编程对象，例如可以是.NET Usercontrol,可以是一个WinForm或者WPF对象，也可以是C++实现的其他对象，例如MFC CView……

在一个具体的页面内或者一个XML文档中，可以用如下格式定义一个网格，这里给出一个3行3列的网格结构对应的DOM描述，具体请参考Sunny.index.html

<content2 target="mainClient"> <layout> <g rows="3" cols="3" width="150,100,50" height="200,180,50"> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="nucleus"></g> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> </g> </layout> </content2>

一个没有子节点的网格，可以承载具体的可编程对象，或者运行时承载一个新的网格结构，取决于具体的编程需求。当我们明确需要一个网格可以承载具体对象的时候，我们需要指定对象的“objid”属性。

当前Sunny可以支持几类可编程对象，具体包括：1、.NET UI对象；2、ActiveX控件对象；3、符合一定规范的C++对象；4、Web页面.

对于.NET UI，我们泛指WinForm、Usercontrol、WPF，后续版本将增加UWP支持。一个网格加载.NET UI，objid属性需要该对象的AssemblyQualifiedName，通常情况下，就是“a.b,c”这样的字符串，其中a是对象的NameSpace，b是类名字，c是组件名。

例如，假设我们有一个.NET组件库xxx.dll，其中有一个FormX对象，其NameSpace是“MyNamespace”，那么objid即为“MyNamespace.FormX,xxx”,对应的网格节点如下：

<g objid="MyNamespace.FormX,xxx"></g>

Nested Grid

Nested Grid means that a Grid can be a child grid of another Grid. We allow nested structures to mean that developers can use "lines" to plan the layout of a rectangular structure ... ...

网格嵌套，是网格的一种复合结构，就是说一个具体的格，可以是一个完整的新的网格结构，这种嵌套使得布局更加具有层次感。

如下给出的是html页面里面或者xml文档中实际的网格嵌套：一个1行2列的网格与一个3行3列的网格形成的复合嵌套

<content3 target="mainClient"> <layout> <g rows="1" cols="2" width="200,100,"> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> <g rows="3" cols="3" width="150,100,150" height="200,180,150"> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="nucleus"></g> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> </g> </g> </layout> </content3>

选项卡（标签）

如果一个grid元素包含子grid节点，同时这个节点没有“rows”属性，Sunny将其解释为一个“选项卡（标签）”，其选项卡个数就是其子节点个数。

Compound Grid: Tabbed Grid with a Grid as a Child Grid

Compound Grid refers to the "composite structure" formed by various container nodes. We allow Tabbed grid and Grid grid to "composite nest" so that developers can implement a more comprehensive UI layout ...

复合网格是指由各种容器节点组成的“复合结构”。我们允许选项卡网格和网格网格“复合嵌套”，这样开发人员可以实现更全面的UI布局

在Html页面上或者xml文档中，一个具体的复合网格示范如下：

<content5 target="mainClient"> <layout> <g style="23"> <g caption="Nested grid" rows="3" cols="3" width="150,100,150" height="200,180,150"> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="nucleus"></g> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl1,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl"></g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl"></g> </g> <g objid="SunnyCtrl.UserControl2,SunnyCtrl" caption="Second Page"></g> <g objid="SunnyCtrl.UserControl3,SunnyCtrl" caption="Third Page"></g> </g> </layout> </content5>

Multi-Level Grid Layout

Here is how to express a "three-level composition" grid structure on a web page or in XML (in fact, any level of composition can be supported). Because the "tab structure" can have multiple "styles", each specific sub grid can carry a variety of programmable objects. On this basis, a more general and comprehensive layout structure can be formed. Due to the flexible and powerful description mechanism of HTML, the web page presents a dual advantage of both native application and web application.

这里显示的是Web页面上或者xml中如何表达一个“三级复合”网格结构（事实上，可以支持任意级复合）。因为“tab结构”可以有多种“样式”，每个特定的子网格可以承载各种可编程对象，在此基础上，可以形成更加一般化、综合的布局结构。由于HTML灵活而强大的描述机制，使网页呈现出一个兼具原生应用与web应用的双重优势的网页...
